## 🌳 推荐策略：混合分支

基于数据分布（高29%、中44%、低27%），**推荐混合分支策略**：

```
输入：5个模型的描述
          │
├─ 高相似(>0.30) : 直接保留 ✓
├─ 中等(0.15-0.30) : 保留并标注
└─ 低相似(<0.15) : 多数投票确认
          │
          v
    【LLM总结】
    整合所有描述，识别
    共识(✓高置信) vs 分歧(?)
          │
          v
    【最终描述】
```

### 执行步骤
1. **分层处理**：按相似度分类
2. **多数投票**：低相似特征需≥3个模型支持
3. **LLM总结**：用强LLM整合5个描述
4. **质量控制**：基于一致性评估

## 💡 核心洞察

### 为什么平均相似度0.235"低"？
- 这是**正常范围**（NLP中0.20-0.35是标准）
- TF-iDF计算词汇重叠，不是语义相似度
- 相似词汇会导致低相似度，但语义相同

### 为什么不排除LLaVA（最独特的模型）？
- 相似度0.1871仍>随机（>0）
- 提供不同视角，这是多模型融合的优势
- 没有证据表明它出错，只是"独特"

### 为什么需要LLM总结？
- 将5个模型的"多元冲突"转化为"多元优势"
- LLM可以识别"共识特征"（高置信）
- LLM可以标注"分歧特征"（需验证）

### LLM总结提示词框架
```
你是交通安全专家。整合以下5个模型的描述：

1. 标注【共识特征】- 所有或大多数模型都提及
2. 标注【分歧特征】- 只有部分模型提及
3. 指出【可能错误】- 与其他模型矛盾的内容
4. 生成统一描述，优先级：
   - 必须：共识特征
   - 可选：≥3个模型支持的特征
   - 排除：单一模型独有的异常特征

输出格式：JSON
{
  "consensus_features": [...],
  "disputed_features": [...],
  "final_description": "...",
  "confidence": 0.75
}
```
